## 2025-02-19 - Removed redundant O(n) scan in inner loop
**Learning:** `np.max([np.max(h.values()) for h in hist_dict.values()])` was being called inside a nested helper function (`hist_dict_fcn`) that executed multiple times for each histogram plotted. Profiling showed this dominated execution time because it was calculating the global max recursively instead of caching it once.
**Action:** Always look for invariants in nested loops and inner functions. Moved the `_max_value_global` calculation outside the `hist_dict_fcn` to speed up plotting. Remember NOT to use `functools.lru_cache` for `hist_dict_fcn` since it returns deepcopies that are mutated by the caller.
## 2025-02-19 - Removed O(N^2) loops and np.polyfit overhead
**Learning:** `make_style_dict_yaml` iteratively loaded and processed objects in deeply nested lists using redundant lookups like `.to_hist()` multiple times. Calculating metrics like linearity inside list comprehensions inside O(N^2) path lookups is extremely slow. `np.polyfit` also incurs significant overhead for basic 1D linear regression.
**Action:** Always scan through dictionary objects directly instead of probing keys or paths dynamically. Iterate through keys via `ch_dir.keys(cycle=False)`. Calculate metrics in a single flat pass, and replace generic matrix-based math functions like `np.polyfit` with manual explicit calculation using `np.sum` to optimize overhead.
