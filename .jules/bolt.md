## 2025-02-19 - Removed redundant O(n) scan in inner loop
**Learning:** `np.max([np.max(h.values()) for h in hist_dict.values()])` was being called inside a nested helper function (`hist_dict_fcn`) that executed multiple times for each histogram plotted. Profiling showed this dominated execution time because it was calculating the global max recursively instead of caching it once.
**Action:** Always look for invariants in nested loops and inner functions. Moved the `_max_value_global` calculation outside the `hist_dict_fcn` to speed up plotting. Remember NOT to use `functools.lru_cache` for `hist_dict_fcn` since it returns deepcopies that are mutated by the caller.
## 2025-02-19 - Removed np.polyfit overhead in styling dicts and string parsing overhead in loop
**Learning:** `np.polyfit` brings massive generalized SVD computation overhead for simple 1D linear regression on small arrays. Similarly, precomputing argument splits before iterative looping, and avoiding Python string representation of Numpy float values inside tight formatting loops via generic f-strings, yields massive speedups.
**Action:** Replace `np.polyfit(..., 1)` with manual vectorized calculation using basic algebra when processing arrays in tight loops. Use `f"{round(x, 10)}"` over `np.round().astype(str)` when getting Python string representations. Always pull string parsing and comprehensions out of recursive/loop contexts by caching into lookup dicts or lists beforehand.
