## 2025-02-19 - Removed redundant O(n) scan in inner loop
**Learning:** `np.max([np.max(h.values()) for h in hist_dict.values()])` was being called inside a nested helper function (`hist_dict_fcn`) that executed multiple times for each histogram plotted. Profiling showed this dominated execution time because it was calculating the global max recursively instead of caching it once.
**Action:** Always look for invariants in nested loops and inner functions. Moved the `_max_value_global` calculation outside the `hist_dict_fcn` to speed up plotting. Remember NOT to use `functools.lru_cache` for `hist_dict_fcn` since it returns deepcopies that are mutated by the caller.
## 2026-07-14 - Fast yield filtering with .values().sum()
**Learning:** In `plot_postfits.py`, we were repeatedly computing `np.sum(get_hist(key).values())`. Since `get_hist(key)` implicitly deepcopies the hist, fetching it inside the loop and converting it to numpy was extremely slow. By precomputing a keep-set for backgrounds, signals, and projections, and passing `raw=True` to avoid deepcopy, then calling native `.values().sum()`, we achieved ~6x speedup on this logic.
**Action:** Always use `raw=True` when fetching hists for read-only aggregation. Always precalculate set lookup tables outside loop iterations. Prefer native `.values().sum()` instead of `np.sum()` for simple array aggregations.
