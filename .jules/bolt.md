## 2025-02-19 - Removed redundant O(n) scan in inner loop
**Learning:** `np.max([np.max(h.values()) for h in hist_dict.values()])` was being called inside a nested helper function (`hist_dict_fcn`) that executed multiple times for each histogram plotted. Profiling showed this dominated execution time because it was calculating the global max recursively instead of caching it once.
**Action:** Always look for invariants in nested loops and inner functions. Moved the `_max_value_global` calculation outside the `hist_dict_fcn` to speed up plotting. Remember NOT to use `functools.lru_cache` for `hist_dict_fcn` since it returns deepcopies that are mutated by the caller.
## 2025-04-09 - [Visual regression tests and set randomness]
**Learning:** Uproot dictionary iteration provides a consistent iteration order based on the ROOT file schema. The old code used `list(set(sum([lists], [])))` which sorts keys seemingly deterministically (thanks to PYTHONHASHSEED environment variables in CI/CD potentially), however, any code replacement that attempts to avoid flattening the lists into a single set, e.g. using `list(dict.fromkeys(...))` to preserve order, ends up breaking the highly sensitive visual regression test suites due to the subtle stack ordering differences that ensue from iterating across differently ordered lists.
**Action:** Always avoid replacing `list(set(...))` or list accumulation techniques in Python when they drive the visual order of elements in matplotlib stacks without extensive visual test verification. Generator expression improvements across `sum()` and `any()` provide excellent, low-risk performance gains in these scenarios.
