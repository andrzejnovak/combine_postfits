**Learning:** Calculating metrics (yield, linearity) using key-driven, top-down path lookups via Uproot inside deep loops incurs extreme O(N^2) overhead from redundant file I/O operations and path parsing. **Action:** Optimize dictionary generation by inverting loops to be directory-driven: retrieve the target Uproot directory (`fitDiag["path/to/dir"]`) exactly once, quickly parse items using `keys(cycle=False)`, and extract objects into `.to_hist()` exactly once to compute all metrics in a single pass. Replace `np.polyfit` on small inputs with direct manual O(N) vectorized slope/intercept equations using `np.sum` and `np.mean`.
