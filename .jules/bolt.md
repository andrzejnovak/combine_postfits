## 2025-02-19 - Removed redundant O(n) scan in inner loop
**Learning:** `np.max([np.max(h.values()) for h in hist_dict.values()])` was being called inside a nested helper function (`hist_dict_fcn`) that executed multiple times for each histogram plotted. Profiling showed this dominated execution time because it was calculating the global max recursively instead of caching it once.
**Action:** Always look for invariants in nested loops and inner functions. Moved the `_max_value_global` calculation outside the `hist_dict_fcn` to speed up plotting. Remember NOT to use `functools.lru_cache` for `hist_dict_fcn` since it returns deepcopies that are mutated by the caller.

## 2025-03-07 - Combine multiple list comprehensions into a single generator pass
**Learning:** `make_style_dict_yaml` had a massive O(n) performance hit by calling `.to_hist()` inside multiple `[ sum(fitDiag[...].to_hist()) for ... ]` list comprehensions for both calculating yields and finding linearity over the same data. Repeated path-based indexing `fitDiag[path]` also proved extremely slow for large ROOT files compared to single object traversal `dir_obj[key]`.
**Action:** When dealing with Uproot directories and extracting information from histograms (like yields and metrics), map available keys directly and perform a single loop over `dir_obj[key]`. Call expensive class conversions like `.to_hist()` EXACTLY ONCE, gather all necessary metrics, and compute the final aggregations cleanly later. Avoid nested list comprehensions when the identical object requires costly IO/parsing for different calculations.
