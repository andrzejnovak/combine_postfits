## 2024-05-02 - O(N) extraction for Uproot dict loops
**Learning:** Computing metrics like yield and linearity using key-driven, top-down path lookups is extremely slow for large ROOT files. The previous implementation performed O(K * F * C) path checks and loaded objects/calculated histograms repeatedly. Also, `np.polyfit` has high overhead for simple 1D linear regressions on small arrays due to SVD.
**Action:** The optimal pattern is directory-driven: iterate through available Uproot directories, retrieve keys using `ch_dir.keys(cycle=False)`, extract objects, call `.to_hist()` exactly once, and compute all metrics simultaneously in a single pass to achieve O(N) complexity. Use manual vectorized calculations for simple regressions instead of `np.polyfit`.
