## 2025-02-19 - Removed redundant O(n) scan in inner loop
**Learning:** `np.max([np.max(h.values()) for h in hist_dict.values()])` was being called inside a nested helper function (`hist_dict_fcn`) that executed multiple times for each histogram plotted. Profiling showed this dominated execution time because it was calculating the global max recursively instead of caching it once.
**Action:** Always look for invariants in nested loops and inner functions. Moved the `_max_value_global` calculation outside the `hist_dict_fcn` to speed up plotting. Remember NOT to use `functools.lru_cache` for `hist_dict_fcn` since it returns deepcopies that are mutated by the caller.

## 2024-03-22 - Replaced O(N^2) list flattening with set comprehension
**Learning:** `list(set(sum([lists], [])))` is an O(N^2) operation because `sum` repeatedly allocates and copies new lists. Set comprehensions like `{k for c in lists for k in c}` are significantly faster and achieve the uniqueness requirement implicitly. Additionally, checking for existence across multiple dicts using `not any("key" in c for c in channels)` short-circuits evaluation, bypassing the need to generate the full set.
**Action:** Always avoid `sum(lists, [])` for flattening. Prefer generator expressions, `itertools.chain.from_iterable`, or set/list comprehensions. Use `any` for boolean existence checks instead of computing intersections or unions.
