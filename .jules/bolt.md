## 2025-02-19 - Removed redundant O(n) scan in inner loop
**Learning:** `np.max([np.max(h.values()) for h in hist_dict.values()])` was being called inside a nested helper function (`hist_dict_fcn`) that executed multiple times for each histogram plotted. Profiling showed this dominated execution time because it was calculating the global max recursively instead of caching it once.
**Action:** Always look for invariants in nested loops and inner functions. Moved the `_max_value_global` calculation outside the `hist_dict_fcn` to speed up plotting. Remember NOT to use `functools.lru_cache` for `hist_dict_fcn` since it returns deepcopies that are mutated by the caller.
## 2026-07-09 - Optimized numpy sum operations on arrays and lists
**Learning:** Using `np.sum()` on arrays, and especially on lists of arrays, incurs significant NumPy scalar conversion and dispatch overhead. Constructing a list of arrays just to sum them (e.g., `np.sum([h.values() for ...])`) is extremely inefficient due to unnecessary 2D array memory allocations. Additionally, using `np.round()` for floating-point equality checks on scalars adds further overhead compared to absolute differences.
**Action:** Replaced instances of `np.sum(array)` with `array.sum()`, which delegates directly to C space faster. Replaced list-comprehension sums with native Python `sum()` and generator expressions (e.g., `sum(h.values().sum() for ...)`) to prevent intermediate allocations, which yielded a ~2-3x speedup in signal matching loops. Replaced `np.round(a) == np.round(b)` with `abs(a - b) < 0.5` for faster floating-point comparisons.
