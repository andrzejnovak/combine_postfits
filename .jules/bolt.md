## 2025-02-19 - Removed redundant O(n) scan in inner loop
**Learning:** `np.max([np.max(h.values()) for h in hist_dict.values()])` was being called inside a nested helper function (`hist_dict_fcn`) that executed multiple times for each histogram plotted. Profiling showed this dominated execution time because it was calculating the global max recursively instead of caching it once.
**Action:** Always look for invariants in nested loops and inner functions. Moved the `_max_value_global` calculation outside the `hist_dict_fcn` to speed up plotting. Remember NOT to use `functools.lru_cache` for `hist_dict_fcn` since it returns deepcopies that are mutated by the caller.

## 2025-02-23 - Replaced O(N^2) list flattening with set comprehension and short-circuiting
**Learning:** Using `sum([c.keys() for c in channels], [])` to flatten nested lists of keys from Uproot directory structures is extremely inefficient (O(N^2) time complexity) because it repeatedly reallocates and concatenates lists. Additionally, flattening *all* keys across *all* channels just to check for the existence of a single key (`"total_signal"`) forces unnecessary reading from the ROOT file and creates a huge memory overhead.
**Action:** Always prefer set comprehensions (e.g., `{k for c in channels for k in c.keys()}`) or `itertools.chain.from_iterable` over `sum(..., [])` for flattening lists. For existence checks across multiple collections, use short-circuiting with `any()` (e.g., `any("total_signal" in c for c in channels)`) to stop searching as soon as the key is found, avoiding redundant data fetching and allocations.
